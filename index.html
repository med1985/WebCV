MODiCA

MODiCA was borne from a growing frustration with using the myriad AFP tools available. We found them to be difficult to use and pretty ugly while still not giving you the breadth of information one needs when analysing AFP documents. The biggest problem however is that they weren't opensource, so we couldn't improve them. The AFP world is a small one and dominated by niche propietary software. So we began designing an AFP parser, fully opensource, with an emphasis on designing the API to be flexible and intuitive while still giving clients a powerful parsing mechanism. We put a heavy focus on both JUnit and integration testing to ensure consistency and to maintain a high quality opensource project.
